# TradePilot AI — PLAN.md

## 1. Цель проекта

TradePilot AI — это внутренний AI-инструмент команды для проверки и запуска онлайн-торговли без собственного склада.

Главная цель первого этапа — не строить публичный SaaS, а проверить реальную бизнес-модель:

- можем ли мы брать товары у поставщиков;
- загружать их прайсы;
- быстро готовить карточки товаров;
- считать реальную прибыль;
- создавать заказы;
- понимать, какие товары выгодно продавать;
- готовиться к размещению на маркетплейсах.

Архитектура должна оставаться SaaS-ready, но текущий фокус — internal-first MVP.

## 3. Зафиксированные локальные порты

TradePilot AI должен использовать только эти локальные порты:

Frontend: http://localhost:3010
Backend: http://localhost:8010
Swagger: http://localhost:8010/docs

Порт 8000 не использовать для TradePilot AI, потому что он занят Energy Copilot API.

Порт 8011 не использовать для TradePilot AI.

## 4. Главная логика продукта

Пользователь работает внутри платформы так:

1. Добавляет поставщика.
2. Загружает прайс поставщика.
3. Система импортирует товары.
4. Система не создаёт дубли при повторной загрузке.
5. AI улучшает карточки товаров.
6. Система считает цену продажи, наценку и прибыль.
7. Товары появляются во внутренней витрине.
8. Команда создаёт тестовые/реальные заказы.
9. Система показывает прибыльность.
10. Команда принимает решение, какие товары продавать дальше.

## 5. Главная фишка

TradePilot AI должен превращать сырой прайс поставщика в рабочий торговый инструмент.

Главная цепочка:

Поставщик → Прайс → Очистка → AI-карточка → Цена → Прибыль → Заказ → Аналитика

Цель: чтобы проект всегда открывался одинаково.

Нужно:

### Этап 3 — реальный расчёт прибыли

Нужно считать не только простую наценку, но и реальную экономику.

Добавить поля:

- закупочная цена;
- цена продажи;
- наценка;
- комиссия маркетплейса;
- доставка;
- упаковка;
- прочие расходы;
- итоговая прибыль;
- маржа в процентах.

Результат:

### Этап 4 — настоящий AI для карточек товаров

Сейчас AI работает как mock.

Нужно:

- сохранить MockAIProvider для тестов;
- добавить реальный LLM provider;
- API-ключи хранить только в .env;
- AI должен улучшать:
  - название;
  - описание;
  - категорию;
  - SEO-текст;
  - цену/обоснование цены.

Результат:

AI реально помогает быстро готовить карточки товаров из сырого прайса.

### Этап 5 — подготовка к Kaspi и маркетплейсам

Пока не делаем полную интеграцию.

Сначала нужно:

- поле “маркетплейс”;
- комиссия маркетплейса;
- ручная цена конкурента;
- статус готовности товара к публикации;
- экспорт товаров в CSV/Excel;
- простая проверка: выгодно / невыгодно продавать.

Результат:

Мы можем подготовить товары к размещению на Kaspi или другом маркетплейсе.

### Этап 6 — аналитика для решений

Analytics должна помогать выбирать товары.

Добавить:

- график прибыли;
- график выручки;
- товары с высокой маржой;
- товары с низкой маржой;
- товары с низким остатком;
- товары без описания;
- товары, где цена требует пересчёта.

Результат:

Команда видит, какие товары стоит продавать, а какие нет.

4. Cursor даёт отчёт.
5. Мы решаем: Keep All или Undo.
